Microservice Breakdown & Team Ownership

Each member owns one service end-to-end — interface contract, mock implementation, cloud implementation, and tests. No dependencies block progress.

🔌 Adapter & Integration Service

Member 1 — Data Adapters Lead
Owns
  • All 6 source adapters (LMS, SharePoint, HR, Power BI, Excel, Teams)
  • Transport layer (API, file, queue, OData readers)
  • Mock JSON / CSV fixtures for each source
  • Schema mapping & normalisation rules
Interface (Port)
SourceAdapter.fetch(time_range) → list[Event]
Tech Stack
  • Azure Functions (Python or C#)
  • Microsoft Graph SDK · OData client
  • Azure Service Bus for event delivery

🗄️ Data Foundation Service

Member 2 — Data Engineering Lead
Owns
  • Azure Data Factory pipelines (bronze → silver → gold)
  • Dataverse entity model + schemas
  • Identity resolution engine (cross-system ID match)
  • Power BI semantic model (Gold layer)
Interface (Port)
DataStore.query(entity, filter) → DataFrame
Tech Stack
  • Azure Data Factory · Dataverse · Microsoft Fabric
  • Power BI Premium · semantic model
  • SQLite for local dev mirror

🤖 Agent Orchestration Service

Member 3 — AI / Agent Lead
Owns
  • Orchestrator, Insight, Report Builder, Action, Critic agents
  • Copilot Studio agent definitions + topics
  • MCP server implementations (LMS / HR / PowerBI / Automate)
  • Prompt engineering & evaluation suite
Interface (Port)
Agent.handle(intent, context) → Response
Tech Stack
  • Azure AI Foundry · Copilot Studio
  • Azure OpenAI (GPT-4o) for reasoning
  • Custom MCP servers (Python FastAPI)

🧠 Memory & Intelligence Service

Member 4 — ML / Memory Lead

  • Cosmos DB schema (memory, vectors, knowledge graph)

  • Predictive ML models (Azure ML) — compliance, attrition

Memory.recall(query, type) → Context
Tech Stack
  • Cosmos DB (NoSQL vector + Gremlin graph)
  • Azure OpenAI embeddings (text-embedding-3-large)
  • Azure ML pipelines + endpoints

Member 5 — Output / Automation Lead

  • Power BI REST API · Tabular Model API

Port Contracts — Interface Definitions

The hexagonal pattern works because every boundary is a contract. Code against the interface; mock or cloud implementation can swap freely.

SourceAdapter — Inbound Port

Implemented by LMSAdapter, HRAdapter, SharePointAdapter, etc.

interface SourceAdapter:
    # Fetch raw events from external system
    fetch(time_range: TimeRange) -> list[Event]

    # Health check — is upstream reachable?
    health() -> HealthStatus

    # Schema discovery for new sources
    describe_schema() -> Schema

DataStore — Persistence Port

Bronze, Silver, Gold, Vector, and Memory stores all implement this.

interface DataStore:
    write(entity: str, records: list[Record]) -> int
    query(entity: str, filter: Filter) -> DataFrame
    vector_search(query: str, top_k: int) -> list[Match]
    graph_query(gremlin: str) -> GraphResult

Agent — Orchestration Port

Implemented by Orchestrator, Insight, Report Builder, Action, Critic.

interface Agent:
    handle(intent: Intent, ctx: Context) -> Response
    tools() -> list[MCPTool]
    memory_keys() -> list[str]
    system_prompt() -> str

OutputAdapter — Driven Port

Implemented by TeamsNotifier, PowerBIPublisher, FlowBuilder, etc.

interface OutputAdapter:
    render(spec: OutputSpec) -> Artifact
    deliver(artifact: Artifact, channel: str) -> DeliveryAck
    schedule(spec: OutputSpec, cron: str) -> JobId

Memory — Recall & Learn Port

Cosmos DB-backed in cloud, local JSON in dev.

interface Memory:
    recall(query: str, type: MemType) -> Context
    remember(event: Episode) -> str
    feedback(event_id: str, signal: Signal) -> None
    distill() -> int  # nightly job

Platform — Cross-Cutting Port

Auth, audit, monitoring, cost — every service uses these.

interface Platform:
    authorize(user: User, resource: str) -> bool
    audit(actor: str, action: str, target: str) -> None
    track_cost(query_id: str, tokens: int) -> None
    check_grounding(response: str, sources: list) -> float

Layer-by-Layer Deep Dive

Each layer's responsibilities, local mock implementation, cloud production implementation, and owner.

LAYER
RESPONSIBILITY
LOCAL (Phase 1)
CLOUD (Phase 2+)
External Systems
(source data origin)
The real systems holding learning, workforce, and reporting data — owned by other teams, treated as black boxes.
JSON / CSV fixtures stored in /mock/sources/ simulating each system's response shape.
Live API endpoints — Cornerstone LMS, SAP SuccessFactors HR, SharePoint Online, Power BI Service.
Source Adapters
(translate inputs)
One adapter per source system. Translates external API responses into the internal canonical event format.
Reads from JSON fixtures, returns normalised Event objects. Same code path as cloud.
Azure Functions calling real APIs via OAuth + Microsoft Graph SDK + OData clients.
Transport Layer
(delivery)
Reliable, scalable delivery — HTTP, file, queue, streaming. Decoupled from source semantics.
In-process function calls. Events delivered synchronously.
Azure Service Bus queues + Event Hub streaming + Azure Functions HTTP triggers.
Ingestion Engine
(unify & resolve)
Schema detection, identity resolution across systems, medallion staging (Bronze → Silver → Gold).
Python scripts running ADF logic locally against fixtures. SQLite as Dataverse mock.
Azure Data Factory pipelines feeding Dataverse + Microsoft Fabric medallion zones.
Data Foundation
(model + entities)
Canonical entity model — Employee, Course, Completion, Team, Skill, Compliance. Single source of truth.
SQLite database mirroring the Dataverse schema with seeded sample data.
Dataverse entities + Power BI semantic model (Gold layer) for analytical queries.
Agent Orchestration
(reasoning)
5 specialist agents (Orchestrator, Insight, Report Builder, Action, Critic) with tool access via MCP.
Local Python multi-agent loop using Azure OpenAI dev tenant. MCP servers running on localhost.
Azure AI Foundry hosting agents + Copilot Studio interface + Azure OpenAI for inference.
Memory & Intelligence
(remember & predict)
Episodic, semantic, procedural memory + vector search + knowledge graph + predictive forecasts.
Cosmos DB Emulator with vector + Gremlin support. Embeddings via Azure OpenAI dev tenant.
Azure Cosmos DB (multi-model: vector + graph + NoSQL) + Azure ML endpoints for predictions.
Output Engine
(generate artifacts)
Generates Power BI reports, Power Automate flows, narrative summaries, and custom React UI on demand.
Local Power BI Desktop + JSON flow generator. React components served via local dev server.
Power BI REST API + Power Automate Management API + Power Pages hosting custom UI.
Outbound Adapters
(deliver actions)
Push results to Teams, Outlook, ServiceNow, webhooks. Approval flows where needed.
Console logging + local file output simulating each delivery channel.
Bot Framework (Teams) + Graph API (Outlook) + REST adapters (ServiceNow / Jira / webhooks).
Platform & Auth
(cross-cutting)
Identity, RBAC, row-level security, audit trails, secret management, admin console.
Mock JWT auth + local audit log file. Admin console runs as local Blazor / React app.
Entra ID + Key Vault + Dataverse RLS + Power Pages admin console + Azure DevOps CI/CD.
Observability
(monitor & cost)
Track agent quality, hallucination rate, cost per query, pipeline health, security events.
Python logging to local files + simple Streamlit dashboard reading those logs.
Azure Monitor + App Insights + Log Analytics + custom Power BI ops dashboards.
