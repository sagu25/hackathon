What We Built

An agentic platform for unified learning & development analytics

The Learning Report Agent is an AI-powered analytics platform that ingests learning data from multiple sources (LMS, HR systems, SharePoint, Power BI), stores it in a structured data lake, and lets users query it in natural language via a multi-agent system.

It was built as an MVP (Phase 1) — fully functional locally, using mock data and local storage, designed so every component can be swapped to cloud services (Azure) with minimal code change.

Core Capabilities Delivered

  • 💬 Natural language chat — ask questions about learning data; agents call the right tools and synthesise answers
  • 📊 On-demand reports — Compliance, Learning KPI, and Skill Gap reports generated instantly
  • 📈 Interactive dashboards — Live Plotly charts: completion heatmaps, treemaps, trend lines, scatter plots
  • ⚡ Flow builder — Agent generates Power Automate flow JSON from natural language intent
  • 🧠 Agent memory — Episodic (past queries), semantic (org knowledge), procedural (recipes) — stored and recalled
  • 🔌 MCP tool layer — Pluggable Model Context Protocol servers for LMS, HR, Power BI, and Power Automate data
  • 🗄️ Bronze/Silver/Gold data lake — Medallion architecture in SQLite with pre-computed KPI aggregates
Current Status

Running locally at http://localhost:8502 with 500 mock employees, 3,732 LMS enrollments, 2,211 skill assessments, and Gold KPIs pre-computed. Azure OpenAI is optional — the app runs with a mock fallback if not configured.

Architecture Overview

7-tier agentic platform — each tier is a logical layer with clear responsibilities

The architecture follows a 7-tier layered design based on the challenge brief. In the MVP, all tiers run in a single Python process. In production, each tier becomes an independent Azure service.

Tier 1 — User Interface & Dashboard

  • Streamlit multi-page app (ui/)
  • Chat Agent page with conversation history
  • Reports page — on-demand generation
  • Dashboards — Plotly charts from Gold layer
  • Flow Library — view/trigger PA flows
  • Admin Console — setup, config, data explorer

Tier 2 — Ingestion Layer

  • Mock data generator with Faker (scripts/generate_mock_data.py)
  • CSV connectors for LMS, HR, courses, skills, budget
  • Database setup with Bronze/Silver/Gold schema (scripts/setup_db.py)
  • Identity resolution via SQL joins

Tier 3 — MCP Tool Layer

  • 4 MCP servers: LMS, HR, Power BI, Power Automate
  • Each server exposes typed tool functions
  • Tools are called by agents via function-calling
  • All backed by the same SQLite database

Tier 4 — Multi-Agent Orchestration

  • Orchestrator agent with Azure OpenAI GPT-4o
  • Intent parsing and tool routing
  • Multi-turn conversation history
  • Agentic loop: up to 6 tool-call cycles per query

Tier 5 — Memory & Intelligence

  • Episodic memory: past queries stored in SQLite
  • Semantic search: ChromaDB vector embeddings
  • Knowledge graph: NetworkX (people→skills→roles)
  • Procedural recipes: default workflows in SQLite

Tier 6 — Generative Output

  • Report generator: Compliance, KPI, Skill Gap
  • Flow builder: Power Automate JSON generation
  • Proactive engine: daily scan for compliance risk
  • Narrative engine: AI-written summaries

Tier 7 — Data Store Layer (SQLite Medallion)

  • Bronze: Raw mock data loaded from CSVs (employees, enrollments, skills, budget)
  • Silver: Cleaned, identity-resolved tables (employees, lms_enrollments, skill_assessments, course_catalog)
  • Gold: Pre-aggregated KPIs (gold_dept_learning_kpis, gold_compliance_summary, gold_skill_gap)
  • Vector Store: ChromaDB persistent store at data/chroma/
  • Memory Store: SQLite tables (agent_memory_episodic, agent_memory_procedural, generated_flows, generated_reports)

Tier 1 — UI (ui/)

Built with Streamlit. Each file in ui/pages/ becomes a separate page automatically. The main entry point is ui/app.py which shows KPI scorecards on the homepage.

  • 1_Chat_Agent.py — Full chat interface. Sidebar has 8 suggested queries. Uses st.session_state to maintain conversation history across turns. After each response, expands to show which tools were called and with what arguments.
  • 2_Reports.py — Dropdown to select report type and department. Generates reports from output/report_generator.py and renders them as Pandas DataFrames + Plotly charts. Reports can be saved to the database.
  • 3_Dashboards.py — Queries the Gold layer directly and renders 6 charts: horizontal bar, scatter bubble, grouped bar, treemap, time-series line, compliance heatmap, and skill gap bar.
  • 4_Flow_Library.py — Shows saved flows from the DB. Has a form to create new flows manually and buttons to use 4 pre-built templates.
  • 5_Admin.py — Has 4 tabs: Setup (runs scripts), Configuration (shows .env status), Agent Memory (shows episodic/procedural memory tables), Data Explorer (SQL table browser).

Tier 2 — Ingestion (scripts/)

Two scripts handle all data initialisation:

generate_mock_data.py

Uses the Faker library (seeded for reproducibility) to generate:

  • 500 employees across 8 departments with realistic names, roles, managers, locations, hire dates
  • 3,700+ LMS enrollments — required courses have 95% enroll rate; optional courses 20–70%
  • Completion status (Completed/In Progress/Not Started) with realistic score distributions
  • 2,200+ skill assessments with self-rated and manager-rated scores for gap analysis
  • Training budget by department (8 rows)
  • 15-course catalog with categories (Compliance, Technical, Leadership, etc.)

setup_db.py

Creates the SQLite database at data/learning_agent.db with these layers:

  • Bronze tables: Raw landing zone (bronze_employees, bronze_enrollments)
  • Silver tables: Cleaned entities — employees, lms_enrollments, course_catalog, skill_assessments, training_budget
  • Gold tables: Pre-aggregated — gold_compliance_summary, gold_dept_learning_kpis, gold_skill_gap
  • Agent memory tables: agent_memory_episodic, agent_memory_procedural, generated_flows, generated_reports

Tier 3 — MCP Servers (mcp_servers/)

Each server is a FastMCP application. Tools are decorated with @mcp.tool() and have typed parameters. In this MVP they run in-process; in production each would be a separate HTTP microservice.

lms_server.py — 6 tools

  • get_completions(department, course_id, status) — filtered LMS completion records joined with employee data
  • get_enrollments(employee_id) — all courses for one employee
  • get_course_catalog(category) — course list
  • get_compliance_summary(department) — required-course completion rates
  • get_dept_kpis(department) — full KPI row from Gold layer
  • get_overdue_learners(department) — employees not done with required courses

hr_server.py — 6 tools

  • get_employee_profile(employee_id) — profile + skills + training history
  • get_org_hierarchy(department) — employee tree with manager links
  • get_role_skills(role) — skill distribution for a job role
  • get_workforce_summary(department) — headcount by status
  • get_skill_gap_report(department) — manager vs self-rating divergence from Gold
  • search_employees(name, department, role) — free-text search

powerbi_server.py — 4 tools

  • list_datasets() — available semantic model datasets
  • query_semantic_model(dataset_name, filters) — query Gold layer by dataset name
  • create_report(title, dataset, chart_type) — returns a report definition (mock embed URL)
  • embed_visual(visual_type, dataset, x_axis, y_axis) — visual embed config

automate_server.py — 4 tools

  • list_flows() — saved flows + available templates
  • trigger_flow(flow_id) — simulated flow run
  • create_flow(name, description, trigger_type, actions) — generates PA flow JSON + saves to DB
  • get_flow_from_template(template_id) — instantiate a named template

Tier 4 — Agents (agents/)

llm_client.py

Wraps the Azure OpenAI client. If AZURE_OPENAI_ENDPOINT and AZURE_OPENAI_API_KEY are set in .env, it uses the real API. Otherwise it returns a clear mock message explaining the missing config. Keeps the app fully functional for demos without credentials.

tools_registry.py

All 18 tools are defined here in OpenAI function-calling JSON format (the "type": "function" schema). This is what gets passed to chat.completions.create(tools=...).

tool_executor.py

A single TOOL_MAP dictionary maps tool names to Python functions. When the LLM decides to call a tool, the orchestrator calls execute_tool_call(name, arguments) which looks up and calls the function.

orchestrator.py — The Brain

This is the main agent loop. For each user message it:

  1. Recalls similar past queries from ChromaDB episodic memory
  2. Builds a message list: system prompt + last 10 conversation turns + user message
  3. Calls Azure OpenAI with all 18 tools available
  4. If the model returns tool calls → executes them → appends results → calls LLM again
  5. Repeats up to 6 turns (prevents infinite loops)
  6. Returns the final answer + saves episode to memory

Tier 5 — Memory (memory/)

agent_memory.py

Three memory types:

  • Episodic: Every query+response saved to SQLite agent_memory_episodic. Also indexed into ChromaDB episodic_memory collection for semantic recall. The recall_similar_episodes(query) function does vector search to find past relevant conversations.
  • Semantic (org knowledge): Each department's Gold KPI row is embedded and stored in ChromaDB org_knowledge collection. Called once on first use. Enables NL grounding — the agent knows org facts without always querying the DB.
  • Procedural: Default "recipes" in SQLite — steps for compliance_report, skill_gap_analysis, weekly_briefing workflows.

knowledge_graph.py

Builds a NetworkX DiGraph from the Silver layer on first call (cached). Nodes: employees, departments, skills. Edges: belongs_to, has_skill, manages. Provides: employees_with_skill(), skill_gap_for_department(), shortest_skill_path(employee_id, target_skill), get_team_under_manager().

Tier 6 — Output (output/)

report_generator.py

Three report builders that query the database and return structured dict objects (not HTML — the UI renders them with Plotly):

  • generate_compliance_report(department) — compliance summary + overdue learners + at-risk departments
  • generate_learning_kpi_report(department) — full KPI breakdown + top/bottom performers
  • generate_skill_gap_report(department) — gap analysis with overrated/underrated skills

proactive_engine.py

daily_scan() queries the Gold layer for depts below 80% compliance threshold and teams with more than 5 overdue learners. Returns an alerts list with severity levels. get_manager_briefing(manager_id) generates a personalised team summary.

Data Flow

How data moves through the 7 tiers end-to-end

Ingestion Flow (one-time setup)

# Step 1: generate_mock_data.py
Faker library → employees.csv, lms_enrollments.csv, skill_assessments.csv
                training_budget.csv, course_catalog.csv

# Step 2: setup_db.py
CSV files → pd.read_csv() → df.to_sql() → SQLite Silver tables
Silver tables → SQL aggregation queries → SQLite Gold tables
Gold KPIs → text serialised → ChromaDB.add() → org_knowledge vector collection

Query Flow (every chat message)

# User types: "What is Engineering compliance rate?"

User message
  → orchestrator.run()
    → recall_similar_episodes()           # ChromaDB similarity search
    → Build messages: [system_prompt + history + user_message]
    → llm_client.chat(messages, tools=ALL_TOOLS)   # Azure OpenAI call
      → LLM decides: call get_dept_kpis(department="Engineering")
    → tool_executor.execute_tool_call("get_dept_kpis", args)
      → mcp_servers/lms_server.get_dept_kpis()
        → db_tools.query_one() → SQLite Gold table → dict
    → Tool result appended to messages
    → llm_client.chat(messages)              # Second LLM call with data
      → LLM generates: "Engineering has 84.2% compliance rate..."
    → memory.save_episode()                  # SQLite + ChromaDB
  ← Returns: {response, tool_calls_made, session_id, intent}

MVP vs Production — Gap Analysis

What is built today and what is needed for the production microservice architecture

The target production architecture (learning_report_agent_microservice_plan.html) uses a Hexagonal / Ports & Adapters pattern with 6 independently deployable Azure microservices. Here is where we stand:

Layer Current MVP (Local) Production Target (Azure) Status
Architecture pattern Monolithic Python app — all in one process Hexagonal/Ports & Adapters — 6 independent microservices with formal interface contracts Partial
Data sources CSV/SQLite mock files generated by Faker Live API connectors: Cornerstone LMS (REST+OAuth), SAP SF HR (OData), SharePoint (Graph API), Workday Phase 2
Inbound adapters Direct DB calls inside MCP servers 6 Azure Functions: LMSAdapter, SharePointAdapter, HRAdapter, PowerBIAdapter, ExcelAdapter, TeamsAdapter Phase 2
Transport / messaging Synchronous in-process function calls Azure Service Bus queues + Event Hub streaming for async, decoupled delivery Phase 2
Ingestion / medallion Python scripts loading CSVs to SQLite Bronze/Silver/Gold Azure Data Factory pipelines → Microsoft Fabric → Dataverse entities (Bronze→Silver→Gold) Phase 2
Agent orchestration Single Python orchestrator, Azure OpenAI function calling, local MCP servers Azure AI Foundry multi-agent hosting + Copilot Studio conversational interface Phase 2
Memory — vector store ChromaDB (local persistent files) Azure Cosmos DB with vector index (DiskANN) via Azure AI Search Phase 3
Memory — knowledge graph NetworkX in-memory graph rebuilt on every startup Cosmos DB Gremlin API — persistent, queryable graph with Gremlin traversal language Phase 3
Predictive ML Stub only — no real model Azure ML pipelines: compliance forecaster, skill gap projector, attrition risk model Phase 3
Power BI output Simulated — returns mock embed URLs and report definitions Power BI REST API + Tabular Model API to programmatically create/publish real reports Phase 2
Power Automate output Generates flow JSON saved to SQLite — not deployed to PA Power Automate Management API — actually creates and activates flows in the tenant Phase 2
Outbound channels None — output is only in the web UI TeamsNotifier (Bot+Adaptive Cards), OutlookAdapter (Graph API), WebhookAdapter, ServiceNow ticketing Phase 2-3
Authentication None — open local app Entra ID SSO + MFA, RBAC roles, Power BI Row-Level Security enforced per user Phase 2
Audit & compliance None Immutable audit log per action, Microsoft Purview data lineage, GDPR/SOC2 reports Phase 4
Observability Streamlit logs only Azure App Insights, per-query cost tracking, hallucination/grounding scores, pipeline health dashboard Phase 4
UI framework Streamlit (fine for MVP/demos) React + Fluent UI (Teams-native), Power Pages admin console, embedded Power BI visuals Phase 4
CI/CD None — manual run Azure DevOps pipelines, per-service independent deploy, prompt versioning + eval regression suite Phase 4
Key Design Principle Already Established

The MVP was intentionally built so each layer communicates via function calls that mirror the target interface contracts. Swapping SQLite for Dataverse, ChromaDB for Cosmos DB, or mock adapters for real Azure Functions requires changing only the implementation behind the interface — not the agent or orchestration logic.

Setup Guide — New Laptop / Fresh Machine

Complete step-by-step instructions to run the project from scratch on any Windows or Mac machine

Before you start — what you need
  • Windows 10/11 or macOS 12+ or Ubuntu 20.04+
  • Internet connection (to download packages)
  • ~500 MB free disk space
  • Azure OpenAI access (optional — app works without it in mock mode)

Windows

Download the installer from python.org/downloads — choose Python 3.11.x.

During installation: ✅ tick "Add Python to PATH" before clicking Install.

Mac

Windows

Download from git-scm.com and install with defaults.

Mac

Troubleshooting Common Issues

ProblemCauseFix
ModuleNotFoundError: No module named 'streamlit' Dependencies not installed or wrong Python environment Run pip install -r requirements.txt again. If using venv, make sure it is activated.
Port 8501 is already in use Another Streamlit or app is on that port Add --server.port 8502 to the run command
Chat Agent says "Azure OpenAI not configured" Missing or wrong .env file Copy .env.example to .env and fill in your Azure credentials. Restart the app.
Dashboards show "No data" or empty charts Database not set up Go to Admin → Setup and click "Generate Mock Data" then "Setup Database". Or run the scripts manually.
UnicodeEncodeError in terminal on Windows Windows console uses CP1252 encoding Run set PYTHONIOENCODING=utf-8 before running, or use Windows Terminal instead of Command Prompt.
chromadb install fails on older machines ChromaDB needs a C++ compiler on some systems Install Visual C++ Build Tools (Windows) or brew install cmake (Mac), then retry pip install.
MCP server import error mcp package version mismatch Run pip install mcp --upgrade
App is very slow on first Chat Agent query ChromaDB vector index being built for the first time Normal on first run — subsequent queries are fast. The index is persisted in data/chroma/.

What the Five Pages Do — Quick User Guide

PageWhat to do
🏠 Home See live KPI scorecards: Active Employees, Total Enrollments, Avg Completion Rate. Links to all pages.
💬 Chat Agent Type any question about learning data. Use suggested queries in the left sidebar for quick demos. Tool calls shown in expandable section below each response. Clear conversation with the sidebar button.
📊 Reports Select report type (Compliance / KPI / Skill Gap) and optionally a department. Click Generate Report. Charts appear automatically. Click Save Report to persist it.
📈 Dashboards Live charts auto-load from the Gold layer. Includes completion bar chart, scatter bubble, budget bar, treemap, trend line, compliance heatmap, skill gap analysis. Filter by department in the skill gap section.
⚡ Flow Library Click "Use template" on any of the 4 pre-built flows to generate them. Expand a flow to see its JSON definition. Click "Run Flow" to simulate triggering it. Or build a custom flow using the form at the top.
⚙️ Admin Setup tab: Generate data + Setup DB buttons with live output. Configuration tab: check Azure OpenAI status. Agent Memory tab: see past queries the agent remembered. Data Explorer tab: browse raw tables with a row limit slider.
